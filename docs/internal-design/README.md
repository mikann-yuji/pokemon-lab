## 記述方針

- 実装の入口になるファイル名、コンポーネント名、関数名を明記する。
- すべての内部関数を網羅するのではなく、仕様理解に必要な主要な構成要素を中心に記述する。
- 現在の実装に基づいて記述し、未実装の理想形は「拡張候補」として分ける。
